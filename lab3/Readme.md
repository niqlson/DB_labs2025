## Детальний опис таблиць

### 1. Таблиця: users
**Призначення:** Зберігання даних користувачів системи

**Структура:**
- id - первинний ключ
- email - обов'язкове поле з валідацією формату
- firstName - обов'язкове поле
- lastName - обов'язкове поле
- verified - статус верифікації
- blocked - статус блокування
- role - роль у системі (ADMIN, USER, MODERATOR)
- password - пароль
- updated_at - час останнього оновлення
- created_at - час створення

**Обмеження:**
- Email повинен містити символ @ та крапку
- Роль може бути тільки ADMIN, USER або MODERATOR

### 2. Таблиця: products
**Призначення:** Каталог товарів магазину

**Структура:**
- id - первинний ключ
- name - назва товару
- image_url - посилання на зображення
- description - опис товару
- available_quantity - доступна кількість (не може бути від'ємною)
- price - ціна (не може бути від'ємною)
- enabled - чи активний товар
- updated_at - час оновлення
- created_at - час створення

### 3. Таблиця: discount
**Призначення:** Управління знижками на товари

**Структура:**
- id - первинний ключ
- product_id - зовнішній ключ до products(id)
- discount_value - розмір знижки (0-100%)
- start_date - початок дії знижки
- end_date - кінець дії знижки (повинен бути не раніше start_date)
- enabled - чи активна знижка
- updated_at - час оновлення
- created_at - час створення

### 4. Таблиця: reviews
**Призначення:** Відгуки користувачів про товари

**Структура:**
- id - первинний ключ
- product_id - зовнішній ключ до products(id)
- user_id - зовнішній ключ до users(id)
- rating - оцінка (1-5 балів)
- text - текст відгуку
- created_at - час створення

### 5. Таблиця: carts
**Призначення:** Кошики покупок користувачів

**Структура:**
- id - первинний ключ
- user_id - зовнішній ключ до users(id)
- created_at - час створення
- updated_at - час оновлення

### 6. Таблиця: cart_items
**Призначення:** Товари в кошиках

**Структура:**
- id - первинний ключ
- product_id - зовнішній ключ до products(id)
- cart_id - зовнішній ключ до carts(id)
- quantity - кількість (більше 0)

### 7. Таблиця: orders
**Призначення:** Інформація про замовлення

**Структура:**
- id - первинний ключ
- user_id - зовнішній ключ до users(id)
- phone_number - номер телефону
- city - місто доставки
- delivery_price - вартість доставки (≥0)
- totalPrice - загальна сума (≥0)
- postal_code - поштовий індекс
- comment - коментар
- payment_type - тип оплати (CARD/CASH)
- status - статус замовлення
- delivery_type - тип доставки
- created_at - час створення
- updated_at - час оновлення

**Обмеження статусів:**
- PENDING - очікує обробки
- CONFIRMED - підтверджено
- SHIPPED - відправлено
- DELIVERED - доставлено
- CANCELLED - скасовано

**Типи доставки:**
- COURIER - кур'єром
- PICKUP - самовивіз
- POST - поштою

### 8. Таблиця: order_items
**Призначення:** Товари в замовленнях

**Структура:**
- id - первинний ключ
- product_id - зовнішній ключ до products(id)
- order_id - зовнішній ключ до orders(id)
- quantity - кількість (більше 0)
- price - ціна на момент замовлення (≥0)

### 9. Таблиця: reservations
**Призначення:** Резервування товарів під замовлення

**Структура:**
- id - первинний ключ
- order_id - зовнішній ключ до orders(id)
- status - статус резерву
- updated_at - час оновлення
- created_at - час створення

**Статуси резерву:**
- PENDING - очікує підтвердження
- CONFIRMED - підтверджено
- CANCELED - скасовано
- COMPLETED - виконано

### 10. Таблиця: reservation_items
**Призначення:** Товари в резервах

**Структура:**
- id - первинний ключ
- product_id - зовнішній ключ до products(id)
- reservation_id - зовнішній ключ до reservations(id)
- quantity - кількість (більше 0)
